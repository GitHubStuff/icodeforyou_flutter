# creature_comforts_service

Shared "last fed" timestamp service backed by Cloud Firestore.

A small Flutter package that exposes a single shared `DateTime` value across
authenticated users via a Firestore document at `shared/last_fed`. Built around
the `LastFedService` interface so programs depend on the abstraction, with a
Firestore-backed implementation provided.

## Features

- `LastFedService` interface returning `Either<LastFedFailure, T>` (no
  exceptions escape the service)
- Sealed `LastFedFailure` hierarchy for exhaustive error handling at call sites
- `read()` — one-shot fetch of the current timestamp
- `watch()` — realtime stream of timestamp changes across all users
- `update({DateTime? occurredAt})` — write the current (or supplied) timestamp
- Firestore-backed implementation (`LastFedServiceFirestore`) with injectable
  `FirebaseFirestore`, `FirebaseAuth`, time source, and error mapper for full
  testability
- Auth-gated: every operation short-circuits with `LastFedUnauthenticated`
  when no user is signed in, before any network call
- All values stored as integer milliseconds since epoch (UTC)

## Getting started

This package depends on `firebase_core`, `firebase_auth`, and `cloud_firestore`.
The consuming program is responsible for:

1. Calling `Firebase.initializeApp(options: DefaultFirebaseOptions.currentPlatform)`
   at startup (typically generated by `flutterfire configure`).
2. Signing a user in via `firebase_auth` before invoking any service method.
3. Registering `LastFedServiceFirestore` with the services broker.

The backend document must exist at `shared/last_fed` with a single integer
field `occurred_at`. Firestore security rules should restrict access to
authenticated users:

```text
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /shared/last_fed {
      allow read, write: if request.auth != null;
    }
  }
}
```

## Usage

```dart
import 'lib/creature_comforts_service.dart';

final LastFedService service = LastFedServiceFirestore();

// Read once
final result = await service.read();
result.match(
  (failure) => print('Read failed: $failure'),
  (when) => print('Last fed at: $when'),
);

// Watch for changes
service.watch().listen((either) {
  switch (either) {
    case Left(value: final failure):
      print('Watch failure: $failure');
    case Right(value: final when):
      print('Updated: $when');
  }
});

// Record "fed now"
await service.update();

// Record an explicit timestamp
await service.update(occurredAt: DateTime.utc(2025, 6, 15, 12, 30));
```

Handle every failure case via exhaustive `switch` on the sealed hierarchy:

```dart
String describe(LastFedFailure f) => switch (f) {
      LastFedUnauthenticated() => 'Sign in first.',
      LastFedDocumentMissing() => 'Backend document is missing.',
      LastFedDecodeFailure() => 'Stored value is malformed.',
      LastFedRemoteFailure() => 'Firestore error: ${f.message}',
    };
```

## Testing

The implementation accepts injectable dependencies for unit testing without a
running Firebase backend:

```dart
final service = LastFedServiceFirestore(
  firestore: FakeFirebaseFirestore(),
  auth: MockFirebaseAuth(signedIn: true),
  now: () => DateTime.utc(2025),
);
```

Inject a custom `LastFedErrorMapper` to verify error-path behavior:

```dart
final service = LastFedServiceFirestore(
  firestore: throwingFakeFirestore,
  errorMapper: (e) => const LastFedDecodeFailure(message: 'sentinel'),
);
```
